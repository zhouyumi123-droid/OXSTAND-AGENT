# v1 实施路线

## 你最终想要的形态

最终想做成：

- 一个招生 AI 智能体软件 / 控制台
- 能从资料到脚本、从封面到剪辑、从发布到归档形成完整链路

## 但 v1 不做全部

当前 v1 只做最小可运行闭环。

## v1 的边界

### 先做

1. 资料结构化
2. 招生内容工作流
3. 平台包装输出
4. 审核标准定义

### 暂不做

1. 自动剪辑成片
2. 自动生成高质量封面图
3. 自动发布多平台
4. 数字人自动播报
5. 完整软件 UI

这些不是不做，而是放到 v1 之后。

## v1 交付目标

### 目标 1：学校内容工作流

输入：

- 课程方向
- 学校主题
- 目标人群
- 内容类型
- 资料摘要
- 表达约束

输出：

- 5 个选题
- 1 个最佳选题
- 1 条脚本
- 1 套发布包装

### 目标 2：稳定知识卡

至少形成 4 张招生知识卡：

1. 港澳台联考
2. AP
3. A-Level
4. 学校总览 / 四大优势

### 目标 3：审核规则

至少明确：

- 什么不能说
- 什么容易夸大
- 什么算学校差异化
- 什么算平台可发内容

## v1 之后的优先顺序

### v1.1

新增审核节点：

- 口径审核
- 平台审核
- 风格审核

### v1.2

新增视频生产辅助：

- 分镜建议
- 素材建议
- 剪辑投喂词

### v1.3

新增数字人版本：

- 数字人口播脚本
- 句长控制
- 语气控制

### v1.4

新增 HTML / Web 工作台：

- 选择输入
- 查看输出
- 手动编辑
- 保存结果

## 当前最值得先做的一个动作

不是写软件界面。

而是先把这 4 张知识卡做出来，并让内容工作流在这 4 条主线下都能稳定输出。
